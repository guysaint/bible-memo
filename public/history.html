1단계 · 첫 출발

기획서 한 장으로 앱 전체를 구현

요청: “이 기획서를 바탕으로 성경 암송 PWA를 완성도 높게 만들어줘.”
  • 프로젝트 세팅(React·TypeScript·Vite·Tailwind·Zustand)
  • 홈 · 구절 추가 · 암송 연습(4모드) · 묶음 시험 · 기록 5개 화면
  • 음성 읽기(TTS), 단어 채점, confetti 축하 효과까지 11단계 완성
2단계 · 배포

인터넷에서 바로 접속되는 주소 만들기

요청: “이 앱에 접속할 수 있는 주소를 알려줘.”
  • GitHub Pages로 배포, 휴대폰·PC 어디서나 접속
  • 코드를 올리면 자동으로 빌드·배포되는 파이프라인 구성
3단계 · 사용성 개선

직접 써보며 발견한 불편함 다듬기

요청: 장·절 숫자 입력, 본문 자동 표시, 홈에서 말씀·묶음 선택 등 5가지 수정.
  • 장·절 입력칸을 누르면 바로 새 숫자로 바뀌게 수정
  • 홈에서 지난 묶음·다른 말씀을 눌러 골라보기
  • 빈 칸을 누르면 곧바로 구절 추가 화면으로
4단계 · 저작권을 정직하게

개역개정 본문, 어떻게 합법적으로 쓸까

요청: “책·장·절을 고르면 본문이 자동으로 떴으면 좋겠어. 개역개정으로.”
  • 개역개정은 대한성서공회 저작물이라 함부로 넣지 않도록 안내
  • 본문을 앱에 넣지 않고, 기기 안에만 보관하는 방식으로 설계
  • 책·장·절을 고르면 본문이 자동으로 채워지도록 구현
5단계 · 교인 배포 준비

기록이 사라지지 않게, 어르신도 쉽게

요청: “데이터가 사라지면 곤란해. 그리고 어르신들도 쉽게 쓰게 하고 싶어.”
  • 데이터 저장 위치와 사라질 위험(아이폰 정책 등)을 솔직히 설명
  • ‘백업 내보내기 / 복원’ 기능 추가 — 새 폰에서도 기록 복구
  • 파일 고르기 없이 ‘한 번 탭’으로 본문 설치되도록 준비
6단계 · 정식 허락

대한성서공회 허락 → 본문 앱에 내장

요청: “허락받았어. 출처는 ‘개역개정 ⓒ 대한성서공회’만 표기.”
  • 개역개정 31,088구절을 앱에 정식 내장(오프라인에서도 동작)
  • 앱을 열기만 하면 본문이 자동 설치 — 버튼조차 누를 필요 없음
  • 모든 화면 하단에 ‘개역개정 ⓒ 대한성서공회’ 출처 표기
덕분에 이제 교인들은 앱만 열면 끝! 가장 어려웠던 부분이 가장 간단해졌어요.

시험 점수 버그 수정 + 구절 수정 기능
